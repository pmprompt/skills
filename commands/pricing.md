# /pricing — Pricing Strategy Workflow

Design pricing that captures value and validates willingness to pay.

## What It Does

Chains value-based pricing frameworks, packaging strategy, and validation experiments.

## When to Use

- New product pricing
- Repricing existing product
- Adding new tier or plan
- Unclear what customers will pay

## Workflow

1. **Monetizing Innovation** — Design pricing before product (4 types: Feature Shocks, Minivations, Hidden Gems, Decoys) (uses `monetizing-innovation` skill)

2. **Value Metric** — Identify the metric that scales with customer value (uses `monetizing-innovation` skill)

3. **Packaging Strategy** — Design Good/Better/Best tiers (uses `monetizing-innovation` skill)

4. **Willingness-to-Pay Validation** — Design Van Westendorp or conjoint experiments (uses `monetizing-innovation` skill)

## Usage

```
/pricing [product or feature]
```

**Example:**
```
/pricing API analytics platform for SaaS companies
```

## Output

- Recommended pricing type and value metric
- 3-tier packaging (feature allocation)
- Pricing validation experiment design
- Competitive pricing analysis

## Next Steps After /pricing

- Run willingness-to-pay research
- Test pricing with early customers
- Monitor price elasticity and churn
