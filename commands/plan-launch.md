# /plan-launch — Product Launch Workflow

End-to-end launch planning from positioning to post-launch metrics.

## What It Does

Chains positioning, messaging, launch tactics, and success metrics into a complete go-to-market plan.

## When to Use

- Launching new product or major feature
- Need structured launch checklist
- First time launching and want best practices
- Stakeholder alignment on GTM approach

## Workflow

1. **Positioning** — Confirm competitive positioning and target audience (uses `positioning-canvas` skill)

2. **Launch Strategy** — Choose launch type (soft/limited/full), channels, timeline (synthesizes best practices)

3. **PMF Validation Plan** — Design Sean Ellis test and measurement approach (uses `pmf-survey` skill)

4. **Success Metrics** — Define leading/lagging indicators and targets (uses `okrs` skill)

## Usage

```
/plan-launch [product or feature]
```

**Example:**
```
/plan-launch AI code review tool for engineering teams
```

## Output

- Launch positioning and messaging
- Phase-by-phase launch plan (soft → limited → full)
- PMF measurement approach (40% benchmark)
- Success dashboard (North Star + input metrics)

## Next Steps After /plan-launch

- Execute launch plan
- Run PMF survey at milestones
- `/growth` — Design retention and growth loops
