# /growth — Growth Strategy Workflow

Design self-reinforcing growth loops and retention mechanics.

## What It Does

Chains growth loops, habit formation, and product-led growth to create a compounding growth engine.

## When to Use

- Have initial PMF, need to scale
- Want to move beyond paid acquisition
- Design viral or retention loops
- Build habit-forming product

## Workflow

1. **Growth Loops** — Map self-reinforcing loops where output becomes input (uses `growth-loops` skill)

2. **Hooked Model** — Design habit loops (Trigger → Action → Reward → Investment) (uses `hooked-model` skill)

3. **Product-Led Growth** — Structure self-serve acquisition and activation (uses `product-led-growth` skill)

4. **Hierarchy of Engagement** — Map core action → accruing benefits → mounting loss (uses `hierarchy-of-engagement` skill)

## Usage

```
/growth [product]
```

**Example:**
```
/growth Project management tool for agencies
```

## Output

- 3-5 growth loop designs (viral, content, paid, sales)
- Hooked model breakdown (habit formation)
- PLG motion (acquire → activate → monetize)
- Engagement hierarchy (retention mechanics)

## Next Steps After /growth

- Prioritize loops by feasibility + impact
- Build instrumentation for loop metrics
- Run experiments on top 2 loops
